# 文档维护检查清单

- 责任边界：文档维护流程与检查项。
- 不负责：具体业务实现与代码更改。
- 目录映射：`docs/process/maintenance-checklist.md`。
- 交付物：可执行检查清单。
- 验收指标：更新一致性与 SRP 保持率。

## A) 更新前

- 明确内容归属（Frontend / Backend / Shared / Process）。
- 确认只修改一个“事实来源”文档，跨模块用链接引用。

## B) 更新中

- 保持五段模板：责任边界 / 不负责 / 目录映射 / 交付物 / 验收指标。
- 避免在规范文档中写实现细节（实现细节放代码或模块文档）。
- 保持命名与路径一致性（如 `docs/shared/*` 只放跨端契约）。

## C) 更新后（SRP 快速复核）

- 是否出现“相同业务跨模块重复描述”？
- 是否有“实现细节”落入“规范文档”？
- 是否有新文档缺少五段模板？
- 跨模块是否用链接而非复制内容？

## D) 归档与导航

- 重要变更追加到 `docs/process/decisions.md`。
- 新增/迁移文档更新 `docs/index.md`。
- 迁移旧文档保留“迁移提示”。
